# Design Brief

## Interface Goal

AI City should feel like a compact simulation console: readable, alive, and easy
to inspect. It should not feel like a marketing page.

## First Screen

The first viewport is the playable dashboard:

- top command bar
- city map
- selected citizen inspector
- economy overview
- jobs panel
- event log

## Visual Direction

- Calm light interface with dark text and crisp panels.
- Accent colors for systems: green for food, amber for credits, red for risk,
  teal for AI and city status.
- Map uses a textured raster illustration of a small cozy town: central plaza,
  readable paths, houses, farm, market, workshop, town hall, water edge, gardens,
  and light citizen markers. Avoid abstract blocks or overly busy route systems.
- Cards are compact, with 8px radius or less.
- Controls use sliders, segmented choices, toggles, and icon buttons where
  appropriate.

## Key Screens For Figma

1. Simulation Dashboard
2. Citizen Detail / Personality Editor
3. AI Event Review Modal
4. Design Tokens / Component Template
5. Future: City Setup / Scenario Builder

## Customization UX

The citizen panel should expose editable sliders:

- Morality
- Empathy
- Ambition
- Risk

It should also show a derived behavior label:

- Principled
- Balanced
- Selfish
- Ruthless

The UI should make it clear that personality affects future decisions, not just
cosmetic labels.
