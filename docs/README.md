# Project Docs

This folder keeps the project thinking close to the code.

- `product.md` defines the first playable scope.
- `architecture.md` explains module boundaries.
- `ai-strategy.md` explains how local AI should be used.
- `design-brief.md` defines the first interface direction.
- `roadmap.md` lists incremental build phases.
- `decisions/` stores architecture decision records.
